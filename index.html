¿Qué es un control de versiones?

Un sistema de control de versiones se encarga del registro de los cambios realizados a lo largo del tiempo en uno o un conjunto de archivos (versiones), de forma tal que estos puedan ser recuperados con precisión de acuerdo a las necesidades de los usuarios. A este tipo de procesos se le puede denominar versionamiento y si bien este tipo de actividades son comunes en archivos o código fuente en la industria del desarrollo de software, este es un proceso aplicable a cualquier tipo de archivo.

¿Qué es GIT?

Git es un sistema de control de versiones que nació en el año 2005 producto de las necesidades que afrontaba la comunidad del kernel de Linux de encontrar un mecanismo que les permitiera soportar las actualizaciones y mantenimientos que requería este gran proyecto. Según Git (2021), algunos de los objetivos trazados que debía cumplir este nuevo mecanismo incluían:

  • Velocidad
  • Diseño sencillo
  • Soporte para múltiples desarrollos no lineales
  • Debía ser un sistema totalmente distribuido
  • Debía ser capaz de manejar eficientemente proyectos de gran envergadura

¿Qué es un repositorio?, repositorio locales y remotos.

Un repositorio es un espacio donde se almacenan y gestionan archivos de un proyecto de software u otra forma de desarrollo colaborativo. Puede contener código fuente, recursos, documentos u otros archivos relacionados con el proyecto. Hay dos tipos principales de repositorios: locales y remotos.

  • Repositorio Local: Es una copia del repositorio que está almacenada en tu computadora local. Normalmente, cuando trabajas en un proyecto, clonas (copias) el repositorio remoto en tu máquina local para poder trabajar con los archivos y hacer cambios. Utilizas comandos como git clone para obtener una copia local del repositorio
  • Repositorio Remoto: Es el repositorio centralizado que se encuentra en un servidor o en la nube. Es el repositorio principal donde se almacena el proyecto y al que contribuyen varios desarrolladores. Ejemplos populares de servicios de repositorios remotos son GitHub, GitLab y Bitbucket. Los cambios realizados en el repositorio local se pueden enviar (push) al repositorio remoto y viceversa (pull).

¿Qué es GITHUB?

GitHub es una plataforma en línea donde los desarrolladores pueden almacenar y colaborar en proyectos de software usando Git. Permite gestionar versiones, trabajar en equipo, y facilita la colaboración en proyectos de código abierto y privados.

¿Qué es una rama en GIT y para qué sirve?

Una rama en Git es una línea de desarrollo separada que permite trabajar en nuevas funcionalidades, correcciones de errores o experimentos sin afectar la versión principal del proyecto. Se utiliza para desarrollar y gestionar cambios de manera organizada y paralela, facilitando la integración controlada de nuevas características en el código base.

¿Cuáles son los estados en GIT?

Untracked (No rastreado): Son archivos que existen en el directorio de trabajo pero que Git no está siguiendo. Esto significa que Git no está registrando cambios en estos archivos. Unmodified (No modificado): Son archivos que están en el directorio de trabajo y que Git está siguiendo, pero no han sido modificados desde la última confirmación (commit). Modified (Modificado): Son archivos que han sido modificados en el directorio de trabajo desde la última confirmación. Git los reconoce como archivos que han tenido cambios desde la última vez que se realizaron cambios. Staged (Preparado): Después de modificar un archivo en el directorio de trabajo, puedes prepararlo para que se incluya en el próximo commit. Este estado se denomina staged y los archivos en este estado están listos para ser confirmados.

¿Qué es la terminal, consola o línea de comandos y para qué sirve?

La terminal, consola o línea de comandos es una interfaz de texto que permite interactuar con el sistema operativo mediante comandos escritos por el usuario. A través de la terminal, los usuarios pueden ejecutar programas, administrar archivos, configurar el sistema y realizar diversas tareas sin depender de interfaces gráficas. Funciones principales de la terminal: Ejecución de Comandos: Permite ejecutar comandos específicos del sistema operativo (como crear, mover o eliminar archivos y directorios) y aplicaciones instaladas. Automatización: Facilita la automatización de tareas mediante scripts (archivos con secuencias de comandos) que pueden ejecutarse en lotes. Administración del Sistema: Proporciona acceso directo para administrar usuarios, configuraciones de red, servicios y otras funciones avanzadas del sistema operativo. Programación y Desarrollo: Es comúnmente utilizado por desarrolladores para compilar programas, gestionar repositorios Git, ejecutar pruebas y realizar otras operaciones relacionadas con el desarrollo de software. Acceso Remoto: Permite acceder y administrar sistemas remotos a través de conexiones SSH (Secure Shell).

Escribir el funcionamiento de cada uno de los siguientes comandos:

  • ls: Lista los archivos y directorios en el directorio actual.
  • cd <directorio>: Cambia al directorio especificado.
  • cd .. : Retrocede un nivel en el sistema de archivos.
  • pwd : Muestra la ruta del directorio actual.
  • mkdir <nombre>: Crea un nuevo directorio con el nombre especificado.
  • touch <nombre>: Crea un nuevo archivo vacío con el nombre especificado.
  • rm <nombre>: Elimina el archivo especificado.
  • cp <nombre> <directorio>: Copia el archivo especificado al directorio indicado.
  • mv <nombre> <directorio>: Mueve el archivo especificado al directorio indicado.
